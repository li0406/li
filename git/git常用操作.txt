1、git status 命令用于查看项目的当前状态
2、git add * 添加文件到缓存区，*号代表所有修改的文件
3、git commit "任务版本号+任务名称"，将缓存区内容添加到仓库中
4、git pull origin 分支名，从线上拉取最新修改到本地合并（注意如果有冲突要先解决冲突）
5、git push origin 分支名，推到线上仓库
如果新建分支则继续执行下述命令
6、git checkout master (一定要切换到主分支，如果在上一个分支直接切分支的话，会把上一个分支的修改携带到新的分支 如果上一个分支的内容不需要上线的话，就麻烦了)
7、git checkout -b 新的分支名
8、git pull master  (拉取主分支的代码)
